## 摘要
AI 编码智能体的快速普及正在从根本上将软件开发人员的角色从代码编写者转变为代码审查者。尽管开发人员花费大量时间阅读和理解代码，但这些智能体生成的 Python 代码的语言熟练度和复杂度在很大程度上仍未被探索。本研究调查了 AI 智能体的代码熟练度，以确定开发人员维护其代码所需的技能水平。利用 AIDev 数据集，我们挖掘了 591 个拉取请求，其中包含由三种不同的 AI 智能体生成的 5,027 个 Python 文件，并使用了 pycefr 这一静态分析工具来分析代码，该工具将 Python 构造映射到从 A1（基础）到 C2（精通）的六个熟练度等级。我们的结果表明：AI 智能体主要生成基础级代码，超过 90% 的构造属于 A1 和 A2 类别，而归类为精通（C2）的不到 1%；AI 智能体和人类的拉取请求具有大致相似的熟练度特征；AI 智能体生成的高熟练度代码来自功能添加和错误修复任务。这些发现表明，虽然具有基础 Python 技能的开发人员通常可以理解 AI 生成的代码，但特定任务可能需要高级熟练度才能审查和维护复杂的、由智能体生成的构造。

## Abstract
The rapid adoption of AI coding agents is fundamentally shifting software developers' roles from code authors to code reviewers. While developers spend a significant portion of their time reading and comprehending code, the linguistic proficiency and complexity of the Python code generated by these agents remain largely unexplored. This study investigates the code proficiency of AI agents to determine the skill level required for developers to maintain their code. Leveraging the AIDev dataset, we mined 591 pull requests containing 5,027 Python files generated by three distinct AI agents and employed pycefr, a static analysis tool that maps Python constructs to six proficiency levels, ranging from A1 (Basic) to C2 (Mastery), to analyze the code. Our results reveal that: AI agents predominantly generate Basic-level code, with over 90% of constructs falling into the A1 and A2 categories, and less than 1% classified as Mastery (C2); AI agents' and humans' pull requests share a broadly similar proficiency profile; High-proficiency code by AI agents are from feature addition and bug fixing tasks. These findings suggest that while AI-generated code is generally accessible to developers with basic Python skills, specific tasks may require advanced proficiency to review and maintain complex, agent-generated constructs.

## 2. 论文提出的方法论
*   **核心思想**：借鉴欧洲语言共同参考框架（CEFR）的语言分级理念，利用静态分析工具评估 Python 代码构造的熟练度等级。
*   **关键技术工具**：使用 `pycefr` 静态分析工具。该工具将 Python 代码构造映射为六个熟练度等级：
    *   **A（基础）**：A1（入门，如 `if` 语句）、A2（初级，如嵌套列表）。
    *   **B（独立）**：B1（中级，如 `with` 语句）、B2（中高级，如列表推导式）。
    *   **C（熟练）**：C1（高级，如生成器）、C2（精通，如元类、装饰器类）。
*   **分析流程**：
    1.  **数据提取**：从 AIDev 数据集中提取 AI 智能体提交的 Pull Requests (PRs)。
    2.  **代码重构**：针对每次提交，利用 GitHub API 获取文件提交前的版本，应用 diff 补丁重构出提交后的完整文件（`pycefr` 需完整文件分析）。
    3.  **差异分析**：对比文件提交前后的构造变化，仅统计新增的代码构造，排除历史遗留代码的干扰。
    4.  **分级统计**：运行 `pycefr` 对新增代码构造进行分级统计。

## 6. 论文的主要结论与发现
*   **RQ1 发现**：AI 智能体主要生成基础级代码。超过 **90%** 的代码构造属于 A1 和 A2 级别，精通级（C2）代码占比不足 **1%**。不同智能体之间存在统计显著差异，但效应量很小。
*   **RQ2 发现**：AI 与人类代码的熟练度分布**大致相似**，均以基础代码为主。AI 生成的 C1 级代码略多于人类（1.12% vs 0.85%），而人类生成的 C2 级代码略多于 AI。
*   **RQ3 发现**：包含高熟练度代码的 PR 主要集中在**功能添加** 和 **错误修复** 任务中。其中，“构建”相关任务的 PR 平均包含的高级熟练度代码数量最高。

## 7. 优点
*   **视角新颖**：首次将语言学习中的 CEFR 框架应用于 AI 代码熟练度评估，为理解代码复杂性提供了量化标准。
*   **方法严谨**：通过差异分析仅关注“新增代码”，避免了将项目原有代码计入统计，确保了评估结果的准确性。
*   **实用价值**：研究结果为开发团队分配代码审查任务提供了依据（如高熟练度 PR 需高级开发者审查），具有实际工程指导意义。

## 8. 不足与局限
*   **工具局限性**：`pycefr` 基于静态分析，仅识别代码构造（如是否使用了装饰器），无法衡量代码的逻辑复杂度、算法难度或上下文语义难度。
*   **对象局限性**：仅研究了 Python 语言，结论可能不适用于 Java、C++ 等其他编程语言。
*   **数据偏差**：数据来源于开源社区，可能无法完全代表企业私有环境下的代码特征。
*   **智能体识别**：部分智能体（如 Claude Code）因作者归属问题被排除在分析之外，可能影响结论的全面性。

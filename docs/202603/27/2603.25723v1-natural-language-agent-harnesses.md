## 摘要
智能体的性能日益依赖于\emph{控制套件工程}（harness engineering），然而控制套件的设计通常深埋于控制器代码和特定运行时的惯例中，使其难以作为一个科学对象进行迁移、比较和研究。我们探讨智能体控制套件的高层控制逻辑是否可以外部化为一种可移植的可执行制品。我们提出了\textbf{自然语言智能体控制套件}（Natural-Language Agent Harnesses, NLAHs），以可编辑的自然语言表达控制套件行为；以及\textbf{智能控制套件运行时}（Intelligent Harness Runtime, IHR），这是一个通过显式契约、持久化制品和轻量级适配器执行这些控制套件的共享运行时。在编程和计算机使用基准测试中，我们对运行可行性、模块消融以及从代码到文本的控制套件迁移进行了受控评估。

## Abstract
Agent performance increasingly depends on \emph{harness engineering}, yet harness design is usually buried in controller code and runtime-specific conventions, making it hard to transfer, compare, and study as a scientific object. We ask whether the high-level control logic of an agent harness can instead be externalized as a portable executable artifact. We introduce \textbf{Natural-Language Agent Harnesses} (NLAHs), which express harness behavior in editable natural language, and \textbf{Intelligent Harness Runtime} (IHR), a shared runtime that executes these harnesses through explicit contracts, durable artifacts, and lightweight adapters. Across coding and computer-use benchmarks, we conduct controlled evaluations of operational viability, module ablation, and code-to-text harness migration.

## 6. 论文的主要结论与发现
*   **可行性验证**：NLAHs 能够在共享运行时下成功执行，证明了将控制逻辑外化为自然语言制品是可行的。
*   **行为效应显著**：共享运行时和控制套件逻辑显著改变了智能体的行为（如 Token 消耗、调用次数大幅增加），而不仅仅是解决率的微小波动。这表明它们是真实的行为控制手段。
*   **模块效应集中**：模块改进的效果集中在少数边界案例上，而非均匀提升整体表现。增加结构并不总是带来性能提升，如果中间验证逻辑与最终评估标准不一致，反而可能引入偏差。
*   **迁移优势**：在 OSWorld 任务中，从代码迁移到 NLAH 的实现（47.2%）优于原生代码实现（30.4%）。这主要归功于 NLAH 将可靠性机制从脆弱的 GUI 修复转移到了更持久的文件支撑状态和制品验证上。

## 7. 优点
*   **创新性视角**：首次明确提出将“控制套件”作为独立的科学研究对象进行外化和标准化，超越了单一的提示工程。
*   **框架解耦**：成功将运行时策略与任务族逻辑分离，使得控制套件具有了可移植性和可组合性。
*   **深入的分析**：不仅关注最终结果，还深入分析了过程指标和失败案例（如验证器与评估器的不对齐），揭示了“更多结构不等于更好性能”的重要洞察。

## 8. 不足与局限
*   **样本量限制**：受限于预算，实验仅在基准测试的子集上进行，结论在全量数据上的普适性有待进一步验证。
*   **自然语言的精度问题**：自然语言不如代码精确，某些依赖隐藏状态或特定调度器的机制可能无法从文本中完全恢复。
*   **运行时污染风险**：强大的共享运行时可能会掩盖部分控制套件文本的效果，导致归因困难。
*   **混杂因素**：文本表示可能引入指令显著性、提示长度等混杂因素，影响消融实验的严格因果推断。

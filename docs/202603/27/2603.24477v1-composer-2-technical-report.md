## 摘要
Composer 2 是一个专为智能体软件工程设计的专用模型。该模型展示了强大的长期规划能力和编程智能，同时保持了在交互式使用中高效解决问题的能力。模型分两个阶段训练：首先进行持续预训练以提升模型的知识和潜在编程能力，随后进行大规模强化学习，通过更强的推理能力、准确的多步执行以及在长跨度现实编程问题上的连贯性，来提升端到端编程性能。我们开发了基础设施，支持在与部署模型相同的 Cursor 框架中进行训练，配备同等的工具和结构，并使用与真实问题紧密匹配的环境。为了衡量模型在日益困难的任务上的能力，我们引入了一个源自大型代码库（包括我们自己的代码库）中真实软件工程问题的基准测试。Composer 2 是一个前沿级编程模型，展示了训练强大领域专用模型的过程。在我们的 CursorBench 评估中，该模型的准确率相比之前的 Composer 模型（61.3）有了显著提升。在公共基准测试中，该模型在我们的框架下的 Terminal-Bench 得分为 61.7，SWE-bench Multilingual 得分为 73.7，与最先进的系统相当。

## Abstract
Composer 2 is a specialized model designed for agentic software engineering. The model demonstrates strong long-term planning and coding intelligence while maintaining the ability to efficiently solve problems for interactive use. The model is trained in two phases: first, continued pretraining to improve the model's knowledge and latent coding ability, followed by large-scale reinforcement learning to improve end-to-end coding performance through stronger reasoning, accurate multi-step execution, and coherence on long-horizon realistic coding problems. We develop infrastructure to support training in the same Cursor harness that is used by the deployed model, with equivalent tools and structure, and use environments that match real problems closely. To measure the ability of the model on increasingly difficult tasks, we introduce a benchmark derived from real software engineering problems in large codebases including our own. Composer 2 is a frontier-level coding model and demonstrates a process for training strong domain-specialized models. On our CursorBench evaluations the model achieves a major improvement in accuracy compared to previous Composer models (61.3). On public benchmarks the model scores 61.7 on Terminal-Bench and 73.7 on SWE-bench Multilingual in our harness, comparable to state-of-the-art systems.

## 1. 论文的核心问题与整体含义
*   **研究背景**：随着大语言模型（LLM）的发展，代码生成的应用已从简单的自动补全转向“智能体软件工程”，即模型需自主浏览代码库、执行命令并解决复杂的工程任务。
*   **核心问题**：现有的公共基准测试（如 SWE-bench）与真实世界的软件工程需求存在严重脱节。公共数据集往往任务描述过于明确、范围狭窄（多为 Bug 修复），且存在数据污染风险，无法准确衡量模型在处理模糊需求、大规模重构等真实场景下的能力。
*   **研究动机**：旨在开发一种能够处理真实世界复杂性、具备长期规划能力的专用编码模型，并建立一套能够真实反映模型实用性的评估体系，探索领域专用模型的训练范式。

## 2. 论文提出的方法论
Composer 2 的训练流程分为两个主要阶段，核心思想是**最小化训练与测试环境的差异**。

### 2.1 基础模型选择
*   选择了 **Kimi K2.5** 作为基础模型（1.04T 总参数 / 32B 激活参数的 MoE 模型），基于其在代码库困惑度、编码知识和状态跟踪能力上的优异表现。

### 2.2 持续预训练
*   **目标**：增强模型的编码领域知识和潜在能力，为后续强化学习打下基础。
*   **流程**：
    1.  主要计算阶段：在 32k Token 序列长度上进行大规模训练。
    2.  长上下文扩展：扩展至 256k 序列长度。
    3.  针对 SFT 阶段：在针对性编码任务上进行短时训练。
*   **多 Token 预测（MTP）**：训练额外的 MTP 层以加速推理时的推测解码，采用自蒸馏方法预测主模型的对数分布。

### 2.3 大规模强化学习
*   **环境构建**：在模拟真实 Cursor 会话的环境中进行训练，模型可访问文件系统、Shell、搜索工具和 Web。
*   **算法优化**：
    *   采用异步 RL 框架，解耦训练与推理。
    *   **GRPO 改进**：移除了长度标准化项以避免长度偏差；不使用标准差归一化优势函数，防止微小差异被放大；不掩码超长输出。
    *   **KL 散度估计**：使用 $k_1 = -\log r$ 估计器替代 $k_3$，解决了后者在分布差异较大时方差爆炸的问题。
*   **自我总结**：为了处理长跨度任务，模型在训练中学习生成自我总结来压缩上下文，而非依赖外部提示压缩，这提高了长程连贯性。
*   **奖励机制**：
    *   **非线性长度惩罚**：引入公式 $C_{length}(x) = \frac{(1 + kx)^{1-q} - 1}{k(1-q)}$。该惩罚在简单任务上对额外计算极度敏感，而在复杂任务上允许更多思考，从而优化交互体验。

### 2.4 基础设施优化
*   **MoE 训练内核**：针对 NVIDIA Blackwell GPU 优化，使用 MXFP8 和 NVFP4 精度，开发了自定义 CUDA 内核。
*   **路由重放**：解决 MoE 模型在推理与训练间专家路由不一致的问题，确保梯度计算的准确性。

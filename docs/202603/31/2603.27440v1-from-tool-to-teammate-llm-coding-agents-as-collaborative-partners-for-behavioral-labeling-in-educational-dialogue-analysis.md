## 摘要
辅导对话的行为分析对于理解学生学习至关重要，然而人工编码仍然是一个瓶颈。我们提出了一种方法，让LLM编码智能体自主改进LLM分类器用于标注教育对话的提示词。在每次迭代中，编码智能体在人工标注的验证数据上运行分类器，分析不一致之处，并提出基于理论的提示词修改建议供研究者审查。将该方法应用于涉及三种智能体和三种分类器的四项实验中的659个AI辅导会话，在留存数据上的4折交叉验证证实了实质性的改进：最佳智能体达到了测试集 $κ=0.78$ (SD$=0.08$)，与人类评分者间信度 ($κ=0.78$) 持平，成本约为每个智能体5至8美元。虽然开发集性能达到了 $κ=0.91$--$0.93$，但交叉验证的结果代表了我们要表达的主要泛化结论。迭代过程还揭示了一种未记录的标注模式：人类编码者一致将困惑的表达视为投入而非脱离。超过最优点后的持续迭代导致了性能退化，这强调了进行留存数据验证的必要性。我们发布了所有的提示词、迭代日志和数据。

## Abstract
Behavioral analysis of tutoring dialogues is essential for understanding student learning, yet manual coding remains a bottleneck. We present a methodology where LLM coding agents autonomously improve the prompts used by LLM classifiers to label educational dialogues. In each iteration, a coding agent runs the classifier against human-labeled validation data, analyzes disagreements, and proposes theory-grounded prompt modifications for researcher review. Applying this approach to 659 AI tutoring sessions across four experiments with three agents and three classifiers, 4-fold cross-validation on held-out data confirmed genuine improvement: the best agent achieved test $κ=0.78$ (SD$=0.08$), matching human inter-rater reliability ($κ=0.78$), at a cost of approximately \$5--8 per agent. While development-set performance reached $κ=0.91$--$0.93$, the cross-validated results represent our primary generalization claim. The iterative process also surfaced an undocumented labeling pattern: human coders consistently treated expressions of confusion as engagement rather than disengagement. Continued iteration beyond the optimum led to regression, underscoring the need for held-out validation. We release all prompts, iteration logs, and data.

*   **核心问题**：虽然大语言模型（LLM）已被用于辅助标注，但提示词的开发和优化仍高度依赖研究人员的反复试错，存在效率瓶颈。
*   **整体含义**：本研究提出将LLM编码智能体从被动的“工具”转变为主动的“队友”，使其能够自主迭代优化提示词。这不仅解决了标注瓶颈，还展示了人机协作发现隐性教学知识（如“困惑即参与”）的新范式。

## 2. 论文提出的方法论

*   **核心思想**：构建一个自主循环流程，让编码智能体执行“开发-测试-分析”循环。智能体不仅运行分类器，还能分析预测错误，提出基于理论的提示词修改建议，由研究人员审核后实施。
*   **算法流程**：
    1.  **初始化**：准备已标注的验证集 $D$、初始提示词 $P_0$、分类器 $M$ 和编码智能体 $A$。
    2.  **迭代循环**：
        *   运行分类器 $M$ 使用当前提示词 $P_{i-1}$ 对数据集 $D$ 进行标注。
        *   计算 Cohen's Kappa 值及各类别指标。
        *   若性能连续两次未提升，则停止迭代。
        *   智能体 $A$ 分析分类器预测与人工标注之间的差异。
        *   智能体 $A$ 提出具体的提示词修改建议并记录理由。
        *   研究人员审核修改建议的理论合理性（如否决则跳过）。
        *   更新提示词 $P_i$。
    3.  **输出**：返回验证集上表现最佳的提示词版本 $P^*$。

## 6. 论文的主要结论与发现

*   **性能表现**：最佳智能体在留出测试集上达到 Kappa = 0.78，与人类评分者间信度（0.78）持平。虽然开发集上可达 0.91-0.93，但交叉验证揭示了约 0.15 的过拟合差距。
*   **迭代风险**：持续迭代并不总是正向收益，超过最优点后会导致性能回归，强调了使用留出验证集的必要性。
*   **同源优势**：智能体与分类器属于同一家族时，主要优势在于收敛速度更快，而非性能上限更高。GPT分类器在跨智能体测试中表现出了最高的性能上限。
*   **知识发现**：智能体在错误分析中发现了人类编码员未明确记录的隐性规则——学生表达“困惑”通常被归类为“参与”而非“脱离”，这与“生产性失败”理论相符。

## 7. 优点

*   **方法论创新**：提出了“智能体即队友”的协作模式，实现了从人工试错到自主迭代的转变，极大降低了提示词工程的人力成本。
*   **发现隐性知识**：证明了该方法不仅能优化任务性能，还能通过分析人机差异挖掘出人类专家未显式表达的教学理论。
*   **严谨的验证机制**：不仅关注开发集表现，还通过4折交叉验证揭示了过拟合风险，并探讨了迭代停止的标准，具有很高的实践指导意义。
*   **透明度高**：公开了所有提示词、迭代日志和代码，可复现性强。

## 8. 不足与局限

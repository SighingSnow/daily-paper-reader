## 摘要
autoresearch 代码库使 LLM 智能体能够通过直接编辑训练代码，在无约束搜索空间中搜索最优超参数配置。在给定的固定计算预算和约束条件下，我们使用 autoresearch 作为测试平台，比较了经典超参数优化 (HPO) 算法与基于 LLM 的方法在调优小型语言模型超参数方面的表现。在固定的超参数搜索空间内，CMA-ES 和 TPE 等经典 HPO 方法始终优于基于 LLM 的智能体。然而，在无约束搜索空间中直接编辑训练源代码的 LLM 智能体，尽管仅使用本地托管的开放权重 27B 模型，仍大幅缩小了与经典方法的差距。能够避免内存溢出故障的方法优于具有更高搜索多样性的方法，这表明可靠性比探索广度更为重要。虽然中小型 LLM 难以跨试验追踪优化状态，但经典方法缺乏领域知识。为了弥合这一差距，我们引入了 Centaur，这是一种将 CMA-ES 的内部状态（包括均值向量、步长和协方差矩阵）与 LLM 共享的混合方法。Centaur 在我们的实验中取得了最佳结果，其 0.8B 变体的表现优于 27B 变体，这表明当与强大的经典优化器配合使用时，低成本的 LLM 便已足够。0.8B 模型不足以进行无约束代码编辑，但足以进行混合优化；而对于固定搜索空间方法，扩展至 27B 并未带来优势。使用前沿模型 Gemini 3.1 Pro Preview 进行的初步实验并未缩小与经典方法的差距。代码可在 https://github.com/ferreirafabio/autoresearch-automl 获取。

## Abstract
The autoresearch repository enables an LLM agent to search for optimal hyperparameter configurations on an unconstrained search space by editing the training code directly. Given a fixed compute budget and constraints, we use autoresearch as a testbed to compare classical hyperparameter optimization (HPO) algorithms against LLM-based methods on tuning the hyperparameters of a small language model. Within a fixed hyperparameter search space, classical HPO methods such as CMA-ES and TPE consistently outperform LLM-based agents. However, an LLM agent that directly edits training source code in an unconstrained search space narrows the gap to classical methods substantially despite using only a self-hosted open-weight 27B model. Methods that avoid out-of-memory failures outperform those with higher search diversity, suggesting that reliability matters more than exploration breadth. While small and mid-sized LLMs struggle to track optimization state across trials, classical methods lack domain knowledge. To bridge this gap, we introduce Centaur, a hybrid that shares CMA-ES's internal state, including mean vector, step-size, and covariance matrix, with an LLM. Centaur achieves the best result in our experiments, with its 0.8B variant outperforming the 27B variant, suggesting that a cheap LLM suffices when paired with a strong classical optimizer. The 0.8B model is insufficient for unconstrained code editing but sufficient for hybrid optimization, while scaling to 27B provides no advantage for fixed search space methods. Preliminary experiments with the frontier model Gemini 3.1 Pro Preview do not close the gap to classical methods. Code is available at https://github.com/ferreirafabio/autoresearch-automl.

*   **核心问题**：在大语言模型（LLM）日益流行的背景下，探究基于 LLM 的智能体在超参数优化（HPO）任务中能否超越 CMA-ES、TPE 等经典算法。
*   **研究背景**：Karpathy 提出的 `autoresearch` 展示了 LLM 通过直接编辑代码进行优化的潜力，但随后的研究指出经典 HPO 方法在固定搜索空间下可能更优。这引发了关于 LLM 在 AutoML 领域实际能力的疑问。
*   **研究动机**：经典 HPO 算法缺乏领域知识，而 LLM 虽具备知识但难以在长序列中稳定跟踪优化状态。本研究旨在系统对比两类方法，并尝试通过混合架构结合两者优势，填补现有研究的空白。

## 2. 论文提出的方法论
*   **核心思想**：提出 **Centaur**，一种混合优化方法。该方法将经典优化算法 CMA-ES 的内部状态显式地共享给 LLM，使 LLM 能够基于优化器的当前状态（而非仅依赖历史文本）进行决策，从而结合 CMA-ES 的状态跟踪能力与 LLM 的领域知识。
*   **关键技术细节**：
    *   **状态共享**：CMA-ES 的内部状态包括均值向量 $\mu$、步长 $\sigma$ 和协方差矩阵 $C$。这些参数被转化为自然语言描述传递给 LLM。
    *   **协同机制**：在优化过程中，以概率 $r$（默认 0.3）调用 LLM。LLM 接收 CMA-ES 的提议、内部状态、历史 Top-5 配置及最近 20 次试验结果，并有权覆盖 CMA-ES 的提议。
    *   **双向更新**：无论配置由谁提出，试验结果都会反馈给 CMA-ES 更新其内部状态，确保优化器持续学习。
*   **算法流程**：
    1.  初始化 CMA-ES 和历史记录。
    2.  进入循环，根据概率 $r$ 决定是否调用 LLM。
    3.  若调用 LLM，提取 CMA-ES 状态 ($\mu, \sigma, C$) 并输入 LLM 获取建议配置；否则使用 CMA-ES 生成配置。
    4.  评估配置性能。
    5.  更新 CMA-ES 内部状态和历史记录。

*   **Benchmark**：基于 `autoresearch` 平台构建测试床，支持固定搜索空间和无约束代码编辑两种模式。

*   **计算预算**：每种方法运行 24 小时，使用 3 个不同的随机种子。

*   **实验规模**：共对比 9 种方法，每种方法在 3 个种子下运行，确保了结果的统计显著性（Gemini 3.1 Pro Preview 因成本仅 1 个种子）。

## 6. 论文的主要结论与发现
*   **固定空间表现**：经典 HPO 方法（TPE, CMA-ES）在固定搜索空间内一致优于纯 LLM 方法。LLM 方法甚至有时不如随机搜索，主要原因是中小型 LLM 难以跟踪优化状态，导致 OOM 率高。
*   **无约束空间表现**：直接编辑代码的 LLM 智能体大幅缩小了与经典方法的差距，显示出 LLM 在灵活搜索空间中的潜力，但仍未超越最佳经典方法。
*   **混合方法优势**：Centaur 取得了所有实验中的最佳结果。它结合了 CMA-ES 的收敛稳定性与 LLM 的领域直觉（如对齐硬件友好的 Batch Size）。
*   **模型规模效应**：
    *   **混合优化**：0.8B 参数的 Centaur 优于 27B 版本，说明配合强优化器时小模型足矣。
    *   **代码编辑**：0.8B 模型能力不足，27B 模型是必需的。
*   **关键因素**：避免 OOM（可靠性）比搜索多样性更能预测最终性能。

## 7. 优点
*   **创新性方法**：Centaur 首次显式地将 CMA-ES 的数学状态（$\mu, \sigma, C$）转化为 LLM 可理解的上下文，有效解决了 LLM 状态跟踪弱的问题。
*   **系统性评估**：构建了一个公平且全面的 Benchmark，涵盖了从固定参数到代码编辑的多种场景，并自动化了搜索空间提取过程。
*   **反直觉发现**：证明了在混合架构中，小型 LLM (0.8B) 配合经典优化器即可超越大型 LLM，具有极高的工程实用价值。
*   **深入分析**：不仅关注最终性能，还深入分析了 OOM 率、搜索多样性、配置轨迹等，揭示了 LLM 在 HPO 中的具体行为模式。

## 8. 不足与局限
*   **任务单一性**：实验仅在 `nanochat`（小型语言模型训练）这一特定任务上进行，结论能否推广至大规模模型训练或其他机器学习任务（如 CV）尚不确定。

*   **搜索空间依赖**：虽然固定空间的参数提取是自动的，但参数范围的设定仍依赖人工先验知识。

## 摘要
工业界从业者和学术研究人员经常利用多智能体系统来加速工作，然而这些系统赖以运行的框架并未提供一种简单、统一的机制来可扩展地管理智能体管控套件的关键方面，这不仅影响了单个人机交互的质量，也限制了从业者通过共享智能体基础设施协调实现共同目标的能力。智能体框架虽然支持了日益复杂的多智能体系统，但定义这些智能体能力的行为规范仍然分散在文本指令文件、框架内部配置以及像MCP服务器这样独立于单个智能体定义运行的机制中，导致这些规范难以在团队和项目间共享、进行版本控制或协作维护。借鉴辐射安全领域的ALARA原则（即辐射暴露保持在可合理达到的最低水平），我们将其应用于智能体上下文，引入了一个声明式上下文-智能体-工具（CAT）数据层，该数据层通过相互关联的文件表达，将每个智能体的工具访问权限和上下文范围限定在其角色所需的最小限度，并推出了用于执行该层的 npcsh 命令行工具。由于系统以结构化方式解析并强制执行这些文件，修改智能体的工具列表将产生确定的行为变更，而非模型可能遵循也可能不遵循的建议。我们在涵盖文件操作、网络搜索、多步骤脚本编写、工具串联和多智能体委派等115项实际任务中，评估了参数量从0.6B到35B的22个本地部署模型。基于总计约2500次执行，我们刻画了各模型家族在不同任务类别上的成功表现及其失效环节。

## Abstract
Industry practitioners and academic researchers regularly use multi-agent systems to accelerate their work, yet the frameworks through which these systems operate do not provide a simple, unified mechanism for scalably managing the critical aspects of the agent harness, impacting both the quality of individual human-agent interactions and the capacity for practitioners to coordinate toward common goals through shared agent infrastructure. Agent frameworks have enabled increasingly sophisticated multi-agent systems, but the behavioral specifications that define what these agents can do remain fragmented across prose instruction files, framework-internal configuration, and mechanisms like MCP servers that operate separately from individual agent definitions, making these specifications difficult to share, version, or collaboratively maintain across teams and projects. Applying the ALARA principle from radiation safety (exposures kept as low as reasonably achievable) to agent context, we introduce a declarative context-agent-tool (CAT) data layer expressed through interrelated files that scope each agent's tool access and context to the minimum its role requires, and \texttt{npcsh}, a command-line shell for executing it. Because the system parses and enforces these files structurally, modifying an agent's tool list produces a guaranteed behavioral change rather than a suggestion the model may or may not follow. We evaluate 22 locally-hosted models from 0.6B to 35B parameters across 115 practical tasks spanning file operations, web search, multi-step scripting, tool chaining, and multi-agent delegation, characterizing which model families succeed at which task categories and where they break down across $\sim$2500 total executions.

# 论文总结：ALARA for Agents

## 1. 论文的核心问题与整体含义

*   **核心问题**：当前多智能体系统缺乏统一、可扩展的机制来管理智能体的行为规范。现有的行为定义碎片化地分布在文本指令文件（如 `.cursorrules`）、框架内部代码以及独立的工具服务器（如 MCP）中，导致配置难以共享、版本控制和协作维护。
*   **安全与可靠性隐患**：这种碎片化导致“解释性依从”问题，即模型可能忽略文本指令，从而引发行为不可控。过度授权的工具访问权限增加了提示注入攻击的风险，且随着上下文窗口增长，模型容易“迷失在中间”，遗忘关键指令。
*   **整体含义**：论文借鉴辐射安全领域的 **ALARA 原则**（As Low As Reasonably Achievable，即辐射暴露保持在可合理达到的最低水平），提出将其应用于智能体上下文管理。核心思想是通过结构化的声明式文件，强制将智能体的工具访问权限和上下文限制在其角色所需的最小范围内，从而实现“最小权限上下文工程”，解决行为规范碎片化和安全性问题。

## 2. 论文提出的方法论

*   **核心思想**：构建一个声明式的 **CAT（Context-Agent-Tool）数据层**，通过文件系统结构而非文本提示来强制执行行为约束。系统解析并强制执行这些文件，确保修改工具列表能产生确定的行为变更，而非依赖模型可能忽略的建议。
*   **关键技术组件**：
    *   **Context 文件**：定义团队层级、共享资源以及编排者，负责划定子团队边界。
    *   **NPC 文件**：定义单个智能体，包含名称、指令、模型配置以及 **Jinx 列表**。Jinx 列表既是工具目录也是权限集，不在列表中的工具对智能体不可见，从而在结构上杜绝了越权调用。
    *   **Jinxes（工具定义）**：以 YAML 格式定义的可执行工具或技能。包含名称、描述、输入类型和执行步骤。支持组合性，一个 Jinx 可以调用其他 Jinx，形成有向无环图（DAG）的执行流。
*   **执行机制**：
    *   通过 `npcsh` 命令行工具执行。
    *   工具调用不再依赖模型的原生函数调用能力，而是通过 Jinja 模板渲染，使得即使是不支持原生工具调用的小模型也能执行复杂工作流。
    *   **确定性脚手架**：将 ReAct 循环分解为确定性的执行骨架，模型仅需在单个步骤中进行解释和决策，降低了出错率。

## 6. 论文的主要结论与发现

*   **模型家族差异大于参数规模差异**：不同模型家族间的性能差异超过了家族内的参数规模差异。经过工具使用训练的 4B 模型可以胜过未经过此类训练的 27B 模型。
*   **工具使用是独立能力**：智能体能力与 MMLU（通用智力）呈线性相关（r≈0.8），但存在显著离群值，表明工具使用是一种需要专门训练的独立能力。
*   **工具调用次数预测性能**：平均工具调用次数是预测 Benchmark 得分的最强指标（r≈0.7），优于任务耗时或重试次数。
*   **重试策略的边际效应**：约 80% 的成功任务在首次尝试即完成。重试的价值在不同任务类别中差异巨大：网络搜索任务重试收益高达 20%，而多智能体委派任务仅 3%，且后者常因上下文历史丢失而性能下降。
*   **委派任务最难**：对于所有模型，多智能体委派都是最困难的类别，主要归因于规范失败而非单纯的模型能力不足。

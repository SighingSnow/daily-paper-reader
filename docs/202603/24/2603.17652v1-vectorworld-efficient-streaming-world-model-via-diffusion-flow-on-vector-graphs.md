## 摘要
自动驾驶策略的闭环评估需要超越日志回放的交互式仿真。然而，现有的生成式世界模型在闭环中常因以下原因而退化： 无历史初始化与策略输入不匹配， 多步采样延迟无法满足实时性要求，以及 长视界下累积的运动学不可行性。我们提出了 VectorWorld，这是一种流式世界模型，在推演过程中增量生成以自车为中心的 $64 \mathrm{m}\times 64\mathrm{m}$ 车道-智能体矢量图瓦片。VectorWorld 通过运动感知门控 VAE 生成策略兼容的交互状态，从而将初始化与历史条件策略对齐。它通过无求解器的一步掩码补全实现实时外绘，其中使用了经区间条件 MeanFlow 和基于 JVP 的大步监督训练的边门控关系 DiT。为稳定长视界推演，我们引入了 $Δ$Sim，这是一种物理对齐的非自车（NPC）策略，包含混合离散-连续动作和可微运动学 logit 塑形。在 Waymo open motion 和 nuPlan 上，VectorWorld 提升了地图结构保真度和初始化有效性，并支持稳定、实时的 $1\mathrm{km}+$ 闭环推演（代码：\href{https://github.com/jiangchaokang/VectorWorld}{code}）。

## Abstract
Closed-loop evaluation of autonomous-driving policies requires interactive simulation beyond log replay. However, existing generative world models often degrade in closed loop due to (i) history-free initialization that mismatches policy inputs, (ii) multi-step sampling latency that violates real-time budgets, and (iii) compounding kinematic infeasibility over long horizons. We propose VectorWorld, a streaming world model that incrementally generates ego-centric $64 \mathrm{m}\times 64\mathrm{m}$ lane--agent vector-graph tiles during rollout. VectorWorld aligns initialization with history-conditioned policies by producing a policy-compatible interaction state via a motion-aware gated VAE. It enables real-time outpainting via solver-free one-step masked completion with an edge-gated relational DiT trained with interval-conditioned MeanFlow and JVP-based large-step supervision. To stabilize long-horizon rollouts, we introduce $Δ$Sim, a physics-aligned non-ego (NPC) policy with hybrid discrete--continuous actions and differentiable kinematic logit shaping. On Waymo open motion and nuPlan, VectorWorld improves map-structure fidelity and initialization validity, and supports stable, real-time $1\mathrm{km}+$ closed-loop rollouts (\href{https://github.com/jiangchaokang/VectorWorld}{code}).

## 1. 论文的核心问题与整体含义

*   **研究背景**：自动驾驶策略的闭环评估需要超越简单日志回放的交互式仿真环境。现有的生成式世界模型虽然能合成场景，但在实际部署为在线仿真器后端时面临三大瓶颈。
*   **核心问题**：
    1.  **初始化不匹配**：现有生成器通常输出静态快照，缺乏历史运动信息，导致依赖历史输入的策略出现“冷启动”瞬态（如急动度激增）。
    2.  **采样延迟高**：基于扩散模型的生成器通常需要多步去噪，计算开销大，无法满足实时流式生成的需求。
    3.  **长时域不可行**：微小的运动学违规在短时仿真中可忽略，但在公里级长时域推演中会累积导致系统崩溃。
*   **整体含义**：本文旨在构建一个能够支持实时、长距离、闭环推演的流式世界模型，解决从静态场景生成到动态交互仿真跨越时的关键工程与算法难题。

## 2. 论文提出的方法论

VectorWorld 提出了一种基于矢量图的流式世界模型，核心在于“按需生成”与“物理对齐”。

*   **核心思想**：将仿真环境建模为以自车为中心的滑动窗口矢量图瓦片（$64m \times 64m$）。系统在自车接近边界时触发“外绘”生成新区域，而非预先生成整个地图。
*   **关键技术细节**：
    1.  **交互状态接口**：
        *   利用**运动感知门控 VAE** 编码场景。对于每个智能体，除了静态状态（位置、尺寸等），还引入运动历史编码。
        *   引入门控机制 $g_i$，对于静止车辆抑制运动编码噪声，对于运动车辆保留动力学特征，从而为下游策略提供“热启动”输入。
    2.  **边缘门控关系 DiT (Edge-Gated Relational DiT)**：
        *   骨干网络基于 Transformer (DiT)，处理异构矢量图（车道节点、智能体节点）。
        *   引入**边缘条件化偏置**和**数值门控**机制，在注意力计算中显式建模车道连接性和车-道关系，防止生成过程中拓扑结构断裂。
    3.  **无求解器的一步采样**：
        *   采用 **Interval-Conditioned MeanFlow** 训练目标，支持大步长传输。
        *   引入 **JVP (Jacobian-Vector Product) 监督**，对大步长预测进行一阶校正，公式为 $V_\theta = u_\theta + \Delta \frac{d}{dt}u_\theta$，使得模型能够通过单次前向传播完成高质量生成，无需迭代求解 ODE。
    4.  **物理对齐的 NPC 策略 ($\Delta$Sim)**：
        *   设计混合离散-连续动作头，结合离散词元和连续残差。
        *   引入 **DKAL (Differentiable Kinematic Alignment Loss)** 和推理时的 Logit 塑形，惩罚运动学不可行的动作，确保长时域推演的稳定性。

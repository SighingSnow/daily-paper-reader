## 摘要
多智能体应用通常将复杂任务作为多阶段工作流执行，其中每个阶段均为一次LLM调用，其输出成为后续步骤上下文的一部分。现有的LLM服务系统主要假设具有相同模型副本的同构集群。这种设计忽视了异构部署的潜力，而在异构部署中，不同大小和能力的模型能够在延迟和性能之间实现更精细的权衡。然而，异构性在具有不同吞吐量和性能的模型间调度方面引入了新的挑战。我们提出了Chimera，这是一个用于异构LLM集群上多智能体工作流服务的预测性调度系统，能够同时改善端到端延迟和任务性能。Chimera应用语义路由来估算每个请求针对各模型的置信度分数，预测工作流的总剩余输出长度，并利用正在处理的预测token量估算各模型拥塞情况以进行负载均衡。我们使用多种异构LLM配置，在代码生成和数学推理的代表性智能体工作流上对Chimera进行了评估。在可比设置下，Chimera追踪到了最佳的延迟-性能前沿，与包括vLLM在内的竞争基线相比，端到端延迟降低了1.2--2.4倍，任务性能平均提升了8.0-9.5个百分点。

## Abstract
Multi-agent applications often execute complex tasks as multi-stage workflows, where each stage is an LLM call whose output becomes part of context for subsequent steps. Existing LLM serving systems largely assume homogeneous clusters with identical model replicas. This design overlooks the potential of heterogeneous deployments, where models of different sizes and capabilities enable finer trade-offs between latency and performance. However, heterogeneity introduces new challenges in scheduling across models with diverse throughput and performance. We present Chimera, a predictive scheduling system for multi-agent workflow serving on heterogeneous LLM clusters that jointly improves end-to-end latency and task performance. Chimera applies semantic routing to estimate per-model confidence scores for each request, predicts the total remaining output length of the workflow, and estimates per-model congestion using in-flight predicted token volumes for load balancing. We evaluate Chimera on representative agentic workflows for code generation and math reasoning using multiple heterogeneous LLM configurations. Across comparable settings, Chimera traces the best latency-performance frontier, reducing end-to-end latency by 1.2--2.4$\times$ and improving task performance by 8.0-9.5 percentage points on average over competitive baselines including vLLM.

## 1. 论文的核心问题与整体含义
*   **研究背景**：随着大语言模型（LLM）应用从单一聊天机器人转向多智能体系统，任务通常以多阶段工作流的形式执行。现有的LLM服务系统（如vLLM）大多假设同构集群（即所有副本为相同模型），忽视了异构部署的潜力。
*   **核心问题**：在异构LLM集群中，不同模型在延迟、吞吐量和任务性能上存在差异。现有的调度系统无法解决以下挑战：
    1.  **工作流依赖性**：端到端延迟源于多阶段排队交互，单请求级调度次优。
    2.  **模型选择与拥塞耦合**：选择能力强但速度慢的模型会加剧资源争用，影响整体延迟。
*   **研究动机**：如何设计一种调度系统，能够在异构LLM集群上联合优化多智能体工作流的端到端延迟与任务性能，填补现有系统在异构环境下的空白。

## 2. 论文提出的方法论
Chimera 是一个构建于推理引擎（如vLLM）之上的中间件层，核心思想是**预测性调度**，通过联合优化模型路由、队列管理和负载均衡来实现目标。

*   **核心组件**：
    1.  **语义路由器**：
        *   **功能**：预测每个候选模型对当前请求的置信度分数（即任务成功概率）。
        *   **实现**：基于 ModernBERT-large 微调的 Transformer 编码器，使用二元交叉熵损失训练，输出各模型的独立置信度。
    2.  **长度预测器**：
        *   **功能**：预测工作流的总剩余输出 Token 长度（当前阶段 + 后续阶段），用于确定调度优先级。
        *   **实现**：使用轻量级的分位数随机森林（QRF），在 CPU 上运行。特征包括 Token 计数、工作流状态、模型名称和文本草图（TF-IDF + SVD）。
        *   **策略**：采用“最短总作业优先”（STJF）策略，优先处理预测总长度短的请求，以降低平均延迟。
    3.  **活动监视器**：
        *   **功能**：跟踪每个引擎的“在途预测 Token 量”，用于估算负载。
    4.  **负载均衡器**：
        *   **功能**：结合置信度与负载情况选择目标模型。
        *   **调度算法流程**：
            1.  计算每个模型的预计延迟（基于在途 Token 量和模型解码速度）。
            2.  找出最快模型 $m_{fast}$ 及其延迟 $L_{fast}$。
            3.  按置信度从高到低遍历模型，选择满足以下条件的模型：
                *   负载约束：$L[m] \le (1 + \tau) \cdot L_{fast}$（$\tau$ 为延迟松弛阈值）。
                *   置信度提升：$q[m] \ge q_{fast} + \Delta s$（$\Delta s$ 为置信度边际）。
            4.  若无满足条件的模型，则回退到最快模型。
*   **防饥饿机制**：引入基于老化机制的优先级提升，当请求等待超过阈值时，临时提升其优先级，防止长请求无限等待。
